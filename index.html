Desafio Final CADASTRO DE PESSOA

Resumo:

O projeto foi desenvolvido utilizando a abordagem back end first, utilizando a linguagem de programação .Net em conjunto com a utilização do Swagger para testes iniciais da conexão com o banco de dados SQLServer e de cada endpoint criado.

No desenvolvimento do front end foi utilizado o Angular na versão 16.2, complementado pela biblioteca Angular Material para aprimorar a estilização da interface.

Boas práticas:

Foi utilizado o conceito de delegação de responsabilidade ao repassar funções ao service, evitando um "fat controller" que pode vir a prejudicar a manutenção do código no futuro.

Facilidade de manutenção:

Foi amplamente utilizado o conceito de componentes do Angular para segmentar a criação de diferentes funcionalidades dentro do projeto, facilitando a organização e a manutenção do código.

Funções:

  • Atualizar cadastro.
  • Desativar cadastro.
  • Ativar cadastro.
  • Excluir cadastro.

Validações:

  • Modelo padrão de inserção de email.
  • Verificação de CEP de acordo com API ViaCep.
  • Bloqueio de registro de documentos e e-mails duplicados.
  • Quantidade mínima de caracteres no nome, telefone e documento.
  • Bloqueio de inserção de letras em documento, telefone e CEP.
